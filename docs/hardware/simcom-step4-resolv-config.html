Step 4 - Configure DNS

Overview:​

Configuring /etc/resolv.conf is necessary to specify the DNS (Domain Name System) servers used by the system for domain name resolution. DNS servers translate human-readable domain names (e.g., www.example.com) into IP addresses that computers can understand. This process is essential for accessing websites, sending emails, and other internet-related tasks. However, certain services like NetworkManager may automatically modify /etc/resolv.conf, potentially overriding user configurations. To ensure that the desired DNS servers are consistently used and to prevent unauthorized changes, it's crucial to properly configure and secure /etc/resolv.conf.

Why it is Needed:​

When your device connects to a network, it needs to know which DNS servers to use for domain name resolution. By default, services like NetworkManager may automatically manage /etc/resolv.conf, leading to unpredictable changes in DNS server settings. This can cause issues such as slow internet browsing or inability to access certain websites if the automatically configured DNS servers are unreliable or not preferred.

To ensure stable and consistent DNS resolution, it's necessary to manually specify the desired DNS servers in /etc/resolv.conf and prevent other services from modifying this configuration. Additionally, locking /etc/resolv.conf helps safeguard against unauthorized changes, ensuring that the specified DNS servers remain intact.

Steps to Configure /etc/resolv.conf:​

  1. Prevent NetworkManager from Managing /etc/resolv.conf:

Edit the NetworkManager configuration file /etc/NetworkManager/NetworkManager.conf and add the following lines under the [main] section:

nano /etc/NetworkManager/NetworkManager.conf

Edit the NetworkManager configuration file /etc/NetworkManager/NetworkManager.conf and add the following lines under the [main] section:

dns=none
rc-manager=unmanaged

The complete NetworkManager.conf will look like this after edit.

[main]
plugins=ifupdown,keyfile
dns=none
rc-manager=unmanaged
[ifupdown]
managed=false

[device]
wifi.scan-rand-mac-address=no
  1. Manually Configure DNS Servers:

Remove any existing entries in /etc/resolv.conf and add the following DNS servers.

nameserver 8.8.8.8
nameserver 8.8.4.4
  1. Lock /etc/resolv.conf File:

To prevent other processes from modifying /etc/resolv.conf, set the file's permissions to make it read-only using the following command:

sudo chattr +i /etc/resolv.conf

This command sets the "immutable" attribute on the file, making it impossible to modify or delete without first removing the attribute.

Conclusion​

By following these steps, you ensure that your system consistently uses the specified DNS servers for domain name resolution and prevents unauthorized changes to /etc/resolv.conf, thereby maintaining a stable and secure network configuration.
