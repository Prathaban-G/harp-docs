Step 12 - Create Harp Service

Overview:​

Creating the Harp service (harp.service) is a crucial step in ensuring the continuous operation of the application on the device. This service runs a Python program responsible for fetching data from Modbus, sending data to the MQTT broker, and establishing a VPN client network for accessing the PLC. By configuring the harp.service, you ensure that the application starts automatically upon system boot, providing seamless functionality and reliability.

Why it is Needed:​

The Harp service (harp.service) is needed to automate the execution of the application on the device. By defining the service, you ensure that the application starts automatically without manual intervention, enhancing the device's reliability and usability. Additionally, the service configuration (harp.service) specifies the dependencies and execution parameters necessary for the proper functioning of the application, ensuring smooth operation under varying conditions.

Steps to Create harp.service:​

Navigate to Systemd Directory:​

Open a terminal and navigate to the systemd directory:

cd /etc/systemd/system

Create harp.service File:​

Create a new file named harp.service and open it in a text editor: 

sudo nano harp.service

Paste Service Configuration:​

Copy and paste the following content into the harp.service file:

[Unit]
Description=Harp Service
After=network.target

[Service]
ExecStart=/home/pi/ota/c_ota.sh
Restart=always

[Install]
WantedBy=default.target

Save and Exit:​

Save the changes and exit the text editor.

Enable harp.service to Run at Startup:​

Enable the harp.service to start automatically at system boot:

sudo systemctl enable harp

Start the harp Service:​

Start the harp service to apply the changes immediately:

sudo systemctl start harp.service

Conclusion:​

By configuring the Harp service (harp.service), you have ensured that the application starts automatically upon system boot, providing seamless functionality and reliability. This service plays a crucial role in automating the execution of the application, enhancing the device's usability and ensuring continuous operation. By following these steps, you have effectively integrated the application into the device's startup process, enabling hassle-free operation and maintenance.
