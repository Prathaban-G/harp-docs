Step 8 - Create Serial ID

Overview:​

Creating the serialid.txt file on /home/pi is essential for the proper functioning of the application. This file contains a unique serial ID that serves as a bridge between the device and the Harp portal application. Constructing the serial ID involves extracting the serial ID from the device's CPU information, modifying it according to specific rules, and adding a prefix and suffix to generate the final serial ID.

Why it is Needed:​

The serialid.txt file plays a crucial role in the communication between the device and the Harp portal application. By generating a unique serial ID based on the device's hardware information, you ensure that each device is uniquely identified within the application ecosystem. This facilitates accurate tracking, management, and interaction with individual devices, enhancing the overall functionality and usability of the application.

Steps to Create Serial id:​

Create the serialid.txt File:​

Open a terminal and navigate to /home/pi:

cd /home/pi

Create the serialid.txt file:

touch serialid.txt

Extract Serial ID from CPU Information:​

Run the following command to extract the serial ID from the device's CPU information:

cat /proc/cpuinfo | grep Serial | cut -d ' ' -f 2

Copy the extracted serial ID.

Modify Serial ID:​

Remove specific characters from the serial ID:

100000dfui456 `remove dfui` 100000456

Generate Final Serial ID:​

final_serial_id="555{batch}$serial_id{device-sequence (unit)}"
555 is the batch
serial_id is from previous step
unit_no is sequence of unit

Adjust Serial ID Length:​

Ensure that the final serial ID does not exceed 20 characters, if exceeds remove 0 to adjust.

Write Final Serial ID to serialid.txt:​

Write the final serial ID to the serialid.txt file:

cd /home/pi
nano serialid.txt

Paste the final serial ID.

Conclusion:​

By following these steps, you have successfully created the serialid.txt file on /home/pi with a unique serial ID generated based on the device's CPU information. This file serves as a critical component in the communication between the device and the Harp portal application, ensuring accurate identification and tracking of individual devices within the application ecosystem. This step is essential for the seamless operation and functionality of the application.
