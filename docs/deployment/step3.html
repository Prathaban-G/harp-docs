Step 3: Configuring OpenVPN

Overview:​

In this step, we will make modifications to the default OpenVPN configuration provided by PiVPN. Specifically, we'll add a route directive to ensure proper routing of traffic to a PLC IP address (192.168.3.1), which is in the same subnet as the Linux device. Additionally, we'll adjust the keepalive settings to maintain the VPN connection for 24 hours.

Explanation of Changes:​

Route Directive:​

The route directive specifies a route to be added to the routing table on the VPN server. In this case, we're adding a route for the subnet 192.168.3.0/24 with a netmask of 255.255.255.0. This route ensures that traffic destined for IP addresses within the 192.168.3.0/24 subnet, including the PLC with IP 192.168.3.1, will be correctly routed through the VPN server. This is essential for enabling communication between devices in different subnets.

Push Directive:​

The push directive is used to push a configuration option to the VPN clients when they connect to the server. Here, we're pushing the same route configuration to the clients, ensuring that they also know how to reach the 192.168.3.0/24 subnet via the VPN server. This ensures bidirectional communication between the VPN clients and devices in the 192.168.3.0/24 subnet.

Keepalive Settings:​

The keepalive directive sets parameters for maintaining the VPN connection. The first value (3600) represents the interval in seconds between keepalive messages sent from the server to the client, while the second value (86400) represents the interval in seconds between retries if no response is received. By setting these values to 3600 seconds (1 hour) and 86400 seconds (24 hours) respectively, we ensure that the VPN connection remains active for 24 hours without interruption.

Configuration Steps:​

  1. Navigate to the OpenVPN server configuration file:
sudo nano /etc/openvpn/server.conf
  1. Add the following lines after the client-config-dir /etc/openvpn/ccd line:
route 192.168.3.0 255.255.255.0
push "route 192.168.3.0 255.255.255.0"
keepalive 3600 86400
  1. Save and close the file.

Final configuration will look like this(reference only).

dev tun
proto udp
port 1194
ca /etc/openvpn/easy-rsa/pki/ca.crt
cert /etc/openvpn/easy-rsa/pki/issued/ip-172-31-23-205_9804f05e-cf70-47b5-a5d2-2f42410fdad5.crt
key /etc/openvpn/easy-rsa/pki/private/ip-172-31-23-205_9804f05e-cf70-47b5-a5d2-2f42410fdad5.key
dh none
ecdh-curve prime256v1
topology subnet
server 10.133.11.0 255.255.255.0
# Set your primary domain name server address for clients
push "dhcp-option DNS 8.8.8.8"
push "dhcp-option DNS 8.8.4.4"
# Prevent DNS leaks on Windows
push "block-outside-dns"
# Override the Client default gateway by using 0.0.0.0/1 and
# 128.0.0.0/1 rather than 0.0.0.0/0. This has the benefit of
# overriding but not wiping out the original default gateway.
push "redirect-gateway def1"
client-to-client
client-config-dir /etc/openvpn/ccd
route 192.168.3.0 255.255.255.0
push "route 192.168.3.0 255.255.255.0"
keepalive 3600 86400
remote-cert-tls client
tls-version-min 1.2
 tls-crypt /etc/openvpn/easy-rsa/pki/ta.key
cipher AES-256-CBC
auth SHA256
user openvpn
group openvpn
persist-key
persist-tun
crl-verify /etc/openvpn/crl.pem
status /var/log/openvpn-status.log 20
status-version 3
syslog
verb 3

#duplicate-cn

By making these changes to the OpenVPN configuration, we ensure proper routing of traffic to the 192.168.3.0/24 subnet and maintain a stable VPN connection for 24 hours. This configuration aligns with the requirements of ensuring seamless communication between devices and uninterrupted connectivity.
